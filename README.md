# BANKIR

BANKIR — статическая лендинговая страница для fintech-продукта международных переводов на банковские карты. Проект описывает пользовательский сценарий перевода средств через Visa, Mastercard и UnionPay: выбор валюты, расчет суммы, отображение курса, время доставки и подтверждение действия.

Сайт опубликован на GitHub Pages: https://cr8v-studio.github.io/bankir/

## Назначение проекта

README фиксирует текущее состояние проекта для разработки, поддержки и передачи в работу. Проект сделан как легкая статическая страница без сборщика и фреймворков, чтобы его можно было быстро открыть локально, проверить в браузере и опубликовать через GitHub Pages.

Основные задачи страницы:

- представить BANKIR как сервис международных переводов;
- показать основной продуктовый сценарий через hero-блок с карточкой перевода;
- объяснить ключевые сценарии использования;
- описать простой порядок действий пользователя;
- закрыть базовые вопросы через FAQ;
- показать юридическую информацию и финальный CTA.

## Технологии

Проект использует минимальный фронтенд-стек:

- HTML для структуры страницы;
- CSS для визуальной системы, адаптива и состояний интерфейса;
- JavaScript для интерактивного калькулятора переводов и раскрытия FAQ;
- GSAP, ScrollTrigger и SplitText через CDN для входных, scroll- и текстовых анимаций;
- Lenis через CDN для плавного desktop-скролла;
- GitHub Actions для публикации на GitHub Pages.

Пакетного менеджера и build step нет: сайт остается статическим и публикуется как набор HTML/CSS/JS-файлов.

## Структура страницы

Страница состоит из нескольких крупных секций:

1. `Header` — логотип, основная навигация и CTA.
2. `Hero` — основной заголовок, преимущества и карточка примера перевода.
3. `Scenarios` — сценарии использования продукта.
4. `How it works` — пошаговое объяснение перевода.
5. `FAQ` — ответы на частые вопросы.
6. `Footer` — юридическая информация и финальный призыв к действию.

Такая структура подходит для лендинга fintech-продукта: сначала формируется ценность и доверие, затем объясняется сценарий, после чего пользователь получает ответы на вопросы и финальный CTA.

## Работа калькулятора

Калькулятор в hero-блоке показывает базовый сценарий международного перевода и работает интерактивно:

- по умолчанию выбраны `Россия / RUB` для отправителя и `США / USD` для получателя;
- стартовая сумма отправления — `10 000,00 RUB`;
- пользователь может менять страну отправителя и получателя через dropdown со странами на русском языке;
- флаги стран берутся из папки `assets/flags/`;
- сумма получателя пересчитывается автоматически после изменения суммы или выбранной страны;
- актуальный курс загружается через публичный API Frankfurter;
- в расчет включается комиссия `3%`, а в интерфейсе отображается примечание `*Курс включает комиссию`;
- если курс временно недоступен для выбранной пары валют, карточка показывает аккуратное fallback-состояние;
- ожидаемое время доставки обновляется на основе выбранной валюты и суммы перевода.

Курсы запрашиваются только для выбранной валютной пары и кешируются в рамках текущей сессии страницы, чтобы не делать лишние запросы при повторном выборе той же пары.

## Структура файлов

```text
.
|-- index.html
|-- css/
|   `-- styles.css
|-- js/
|   `-- main.js
|-- assets/
|   |-- flags/
|   |-- icons/
|   `-- images/
|-- .github/
|   `-- workflows/
|       `-- pages.yml
`-- README.md
```

Ключевые файлы:

- `index.html` — разметка всех секций страницы.
- `css/styles.css` — дизайн-система, сетки, адаптив, состояния элементов и motion-настройки.
- `js/main.js` — логика калькулятора переводов, загрузки курсов, выбора стран и раскрытия FAQ.
- `assets/` — SVG-иконки, флаги и графические элементы интерфейса.
- `.github/workflows/pages.yml` — workflow для автоматического деплоя на GitHub Pages.

## Локальный запуск

Проект можно запустить без установки зависимостей:

```bash
python3 -m http.server 4173
```

После запуска открыть в браузере:

```text
http://127.0.0.1:4173/
```

Останавливается сервер сочетанием клавиш `Control + C` в терминале.

## Публикация

Деплой настроен через GitHub Actions. При каждом push в ветку `main` запускается workflow `.github/workflows/pages.yml`, который публикует статические файлы на GitHub Pages.

Публичная ссылка:

```text
https://cr8v-studio.github.io/bankir/
```

Репозиторий:

```

## Правила поддержки

При доработке страницы важно сохранять текущую логику проекта:

- не добавлять фреймворки без необходимости;
- держать HTML семантичным и читаемым;
- использовать существующие CSS-переменные и паттерны;
- сохранять адаптивность на мобильных и desktop-разрешениях;
- не добавлять тяжелые библиотеки для простых UI-состояний;
- проверять, что все SVG и локальные ассеты доступны по относительным путям;
- после изменения коммитить и пушить обновление в `main`.

git status

```
